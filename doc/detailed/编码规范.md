# dingtalk-opencode 编码规范

**版本**：1.0
**技术栈**：TypeScript / Node.js 20+ (ESM)

## 1. 项目结构

```
src/
├── index.ts                   # 入口 + 路由
├── ai-handler.ts              # AI 处理循环
├── opencode.ts                # OpenCode API 客户端
├── dingtalk.ts                # 钉钉消息发送
├── session-store.ts           # 会话持久化
├── message-queue.ts           # 消息排队
├── watchdog.ts                # 健康检测
├── server-manager.ts          # 服务管理
├── project-registry.ts        # 项目配置加载
├── project-context-store.ts   # 项目绑定持久化
├── config.ts                  # 环境变量
├── logger.ts                  # 日志
└── types.ts                   # 类型定义
```

## 2. 项目特有编码约定

### 2.1 分层调用约束

| 调用来源 | 允许调用的模块 | 禁止调用的模块 | 说明 |
|---------|-------------|-------------|------|
| index.ts (路由) | ai-handler.ts, dingtalk.ts, server-manager.ts, project-registry.ts, project-context-store.ts | opencode.ts 直接调用 | 入口仅做路由分发，API 调用通过 ai-handler 编排 |
| ai-handler.ts | opencode.ts, dingtalk.ts, watchdog.ts, session-store.ts | index.ts、server-manager.ts | 处理循环不绕回入口 |
| watchdog.ts | opencode.ts | 其他所有模块 | 看门狗仅依赖健康检查，不耦合业务逻辑 |

### 2.2 并发策略

| 场景 | 策略 | 实现方式 | 来源 |
|------|------|----------|------|
| 消息排队 | Promise 链串行 | `prev.then(fn, fn)` — 无论前序成功/失败都执行 | message-queue.ts |
| AI 处理中断 | AbortController + AbortSignal | 看门狗调用 `controller.abort()` → fetch 抛出 AbortError | watchdog.ts + ai-handler.ts |
| 文件写入防抖 | 2s 延迟定时器 | 多次 set 合并为一次 flush | session-store.ts |

### 2.3 异常处理策略

| 场景 | 处理方式 |
|------|----------|
| OpenCode API 调用失败 | 在 ai-handler.ts 上层捕获，触发重试循环（最多 3 次） |
| 钉钉消息发送失败 | 在 dingtalk.ts 内部 catch，日志 warn，不向上传播 |
| 看门狗检测异常 | 异常吞掉，state 不变（网络抖动不误判） |
| JSON 文件损坏 | 日志 error，以空 Map 启动，不阻塞主进程 |
| 子进程异常退出 | 更新状态为 failed，非强制阻断 |
| 参数验证 | 使用 `throw new TypeError("描述")` 或内联校验后 return |

### 2.4 响应与错误格式

| 场景 | 处理方式 |
|------|----------|
| 成功回复 | Markdown 格式（`buildReplyMessage` 构建） |
| 处理中通知 | 纯文本「⏳ 正在处理中，请稍候...」 |
| 异常提示 | 按类型：服务不可用 / 超时 / 处理失败，由 `sendProcessingError` 分发 |

### 2.5 其他项目特有约定

- 所有模块顶层导出函数，不使用 class（函数式模块，无 DI 容器）
- 跨模块调用直接 import 函数，不通过接口抽象（单进程，无多实现需求）
- 所有配置文件（projects.json、data/*.json）操作使用绝对路径，基于 `process.cwd()`
- 子进程启动继承 `process.env`，不额外传参
