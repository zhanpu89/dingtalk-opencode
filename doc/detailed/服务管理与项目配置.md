## 1. 功能描述

- 功能1：**配置加载** — 从环境变量和 `.env` 文件加载运行时配置，校验必填项，提供默认值
- 功能2：**项目注册表** — 加载、校验、查询 `projects.json` 中的项目声明
- 功能3：**默认服务管理** — 启动/停止/重启默认的 opencode serve 进程
- 功能4：**项目动态服务** — 按需启动/状态查询/复用项目级 opencode serve 子进程
- 功能5：**资源回收** — 空闲超时回收 + 进程退出时统一清理
- 功能6：**强制重启** — 通过钉钉指令停止所有 opencode 进程、持久化会话、执行重启脚本后退出

## 2. 业务规则

| 规则编号 | 规则描述 | 配置来源 |
|----------|----------|---------|
| SVR-REG-01 | 项目 `id` 和 `name` 在 `projects.json` 中必须唯一，重复时启动报错 | 硬编码 |
| SVR-REG-02 | 项目 `path` 必须是绝对路径，启动时经过 `fs.realpath` 解析 | 硬编码 |
| SVR-REG-03 | realpath 解析后的项目路径必须位于 `ALLOWED_PROJECT_ROOTS` 任一前缀之下 | 环境变量_ALLOWED_PROJECT_ROOTS |
| SVR-REG-04 | 动态项目服务端口从 `PROJECT_SERVER_PORT_START` 递增扫描，启动前检查端口是否可监听 | 环境变量_PROJECT_SERVER_PORT_START（默认 4100） |
| SVR-REG-05 | 子进程启动后轮询 `GET /global/health`，最多等待 30 秒，超时标记为 failed | 硬编码 |
| SVR-REG-06 | 子进程异常退出后更新状态为 stopped，下次切换时重新启动 | 硬编码 |
| SVR-REG-07 | 动态项目服务仅监听 127.0.0.1 | 环境变量_PROJECT_SERVER_HOSTNAME（默认 127.0.0.1） |
| SVR-REG-08 | 项目目录可选校验内容：`.git`、`opencode.json`、`AGENTS.md`、`package.json`、`pyproject.toml`、`go.mod` 任一存在即通过；均不存在时给出 warn | 硬编码 |

## 4. 功能逻辑（详细步骤 + 生产级伪代码）

### ServerManager — 启动项目服务

```python
async def startProject(project):
    # 步骤1: 检查是否已有运行实例
    if project_id in running_instances:
        instance = running_instances[project_id]
        if instance.status == "running":
            return instance.baseUrl

    # 步骤2: 分配端口（从 PROJECT_SERVER_PORT_START 开始递增扫描）
    port = find_available_port(start_port)
    # RETRY: 最多尝试 100 个端口

    # 步骤3: 创建 ServerInstance 记录
    instance = ServerInstance(
        id=project.id,
        type="project",
        projectPath=project.path,
        port=port,
        baseUrl=f"http://{hostname}:{port}",
        status="starting",
        startedAt=now()
    )
    running_instances[project.id] = instance

    # 步骤4: spawn 子进程
    child = spawn("opencode", ["serve", "--port", str(port)], {
        "cwd": project.path,
        "env": { ...process.env }
    })
    # LANGUAGE_SPECIFIC: TypeScript - 使用 child_process.spawn()
    instance.process = child

    # 步骤5: 轮询健康检查（最多 30s）
    start_time = now()
    while now() - start_time < 30000:
        try:
            res = await http_get(f"{instance.baseUrl}/global/health")
            if res.ok:
                instance.status = "running"
                instance.lastUsedAt = now()
                return instance.baseUrl
        except:
            pass
        await sleep(1000)

    # 步骤6: 超时 — 终止进程，标记 failed
    child.kill("SIGTERM")
    instance.status = "failed"
    # TIMEOUT: 30s — 健康检查超时
    # FINALLY: 确保进程被终止
    #   1. 发送 SIGTERM
    #   2. 2s 后未退出 → SIGKILL
    #   3. 更新状态为 failed
    raise TimeoutError("服务启动超时")
```

### ServerManager — 停止与清理

```python
def stop():
    # 步骤1: 停止默认服务
    if default_process:
        default_process.kill("SIGTERM")
        # FINALLY: 确保进程退出

    # 步骤2: 停止所有项目子进程
    for instance in running_instances.values():
        if instance.process:
            instance.process.kill("SIGTERM")
            # 等待 2s，未退出则 SIGKILL
        instance.status = "stopped"

    # 步骤3: 清理运行记录
    running_instances.clear()
```

### 强制重启流程

```python
# 用户发送 "强制重启" / "暴力重启"
def forceRestart():
    # 步骤1: 停止所有项目服务
    serverManager.stopAllProjects()

    # 步骤2: 持久化会话数据
    sessions.flush()

    # 步骤3: 检查重启脚本是否存在
    script_path = "scripts/restart_bot.sh"
    if not file_exists(script_path):
        dingtalk.sendTextMessage("重启脚本不存在，请手动执行：npm run start:all")
        return

    # 步骤4: 以 detached 模式启动重启脚本
    child = spawn("bash", [script_path], { detached: true })
    child.unref()

    # 步骤5: 通知用户并退出
    dingtalk.sendTextMessage("✅ 机器人服务正在重启，请稍候...")
    process.exit(0)  # 2s 后退出
```

### ProjectRegistry — 加载与校验

```python
def load(config_path):
    # 步骤1: 读取 JSON 文件
    raw = read_file(config_path)
    projects = json_parse(raw)
    # 必须为数组
    if not Array.isArray(projects):
        raise ValidationError("projects.json 必须为数组格式")

    # 步骤2: 校验每个项目（逐项校验，无效项跳过）
    for p in projects:
        # 必填字段检查
        if not p.id or not p.name or not p.path:
            log.warn(f"跳过项目: 缺少必填字段 id/name/path: {p}")
            continue

        # id 唯一性
        if id_set.has(p.id):
            log.warn(f"跳过项目: 重复 id: {p.id}")
            continue
        id_set.add(p.id)

        # name 唯一性
        if name_set.has(p.name):
            log.warn(f"跳过项目: 重复 name: {p.name}")
            continue
        name_set.add(p.name)

        # path 必须是绝对路径
        if not path.isAbsolute(p.path):
            log.warn(f"跳过项目: 路径非绝对路径: {p.path}")
            continue

        # 目录必须存在
        if not fs.existsSync(p.path) or not fs.statSync(p.path).isDirectory():
            log.warn(f"跳过项目: 目录不存在: {p.path}")
            continue

        # realpath 解析
        real_path = fs.realpathSync(p.path)
        # 校验是否在允许的根目录下
        if not any(real_path.startswith(root) for root in ALLOWED_PROJECT_ROOTS):
            log.warn(f"跳过项目: 路径不在允许范围内: {p.path}")
            continue

        result.append(p)

    # 步骤3: 缓存结果（仅有效项目）
    cached_projects = result
    return result
```

## 5. 状态机与状态流转

### 5.1 服务实例状态枚举

```python
class ServerStatus(enum):
    STARTING = "starting"   # 子进程已 spawn，等待健康检查通过
    RUNNING = "running"     # 健康检查通过，服务可用
    FAILED = "failed"       # 启动超时或进程异常退出
    STOPPED = "stopped"     # 已停止
```

### 5.2 合法状态转换表

| 当前状态 | 目标状态 | 触发操作 | 前置条件 | 副作用 |
|---------|---------|---------|---------|--------|
| — | STARTING | startProject() | 端口可用、路径合法 | spawn 子进程 |
| STARTING | RUNNING | 健康检查通过 | 30s 内 health 返回 200 | 记录实例信息 |
| STARTING | FAILED | 健康检查超时 | 30s 超时 | kill 子进程 |
| RUNNING | STOPPED | stop() | — | kill 子进程 |
| RUNNING | FAILED | 子进程 exit 事件 | exit code ≠ 0 | 记录实例状态 |
| STOPPED | STARTING | startProject() | 重新分配端口 | spawn 新进程 |

## 6. 使用的算法

- **端口分配**：从 `PROJECT_SERVER_PORT_START` 开始递增，检查端口是否可监听（`net.createServer().listen()` → 成功则关闭返回）；最多尝试 100 个端口
- **健康检查轮询**：固定间隔 1s，总超时 30s，遇到第一个 200 OK 即返回
- **重启脚本（`scripts/restart_bot.sh`）**：
  - 延迟 3s 等待父进程退出
  - `pkill -f` 停止所有 opencode / tsx / concurrently 进程
  - 删除 `data/session-map.json`、`data/project-context.json` 清除可能损坏的会话状态
  - `nohup npm run start:all` 重新拉起整套服务

## 11. 安全要求

- **路径安全**：`ALLOWED_PROJECT_ROOTS` 白名单校验 + `fs.realpathSync` 解析，防止符号链接逃逸
- **进程隔离**：子进程继承父进程环境变量，不额外暴露敏感信息
- **端口安全**：动态服务只监听 127.0.0.1，不对外暴露
- **输入限制**：不允许用户通过钉钉传入端口、启动参数或 shell 命令
