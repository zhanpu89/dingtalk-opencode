## 1. 功能描述

- 功能1：**会话映射持久化** — 将「钉钉会话标识 ↔ OpenCode session ID」的映射关系保存到本地 JSON 文件，程序启动时自动加载恢复，写入时 2s 防抖防频繁 I/O
- 功能2：**项目绑定持久化** — 将「钉钉会话 ↔ 项目 ID」的绑定关系保存到本地 JSON 文件，支持按会话查询当前绑定的项目
- 功能3：**会话轮次计数** — 内存级轮次计数器，每轮消息 +1，达到 `MAX_ROUNDS_PER_SESSION` 后触发实例回收重置计数（不持久化，重启即清零）

## 2. 业务规则

| 规则编号 | 规则描述 | 配置来源 |
|----------|----------|---------|
| STO-REG-01 | 会话映射文件 (`data/sessions.json`) 不存在时自动创建空文件 `{}`，禁止启动报错 | 硬编码 |
| STO-REG-02 | 写入操作触发防抖定时器，连续 2s 内的多次 set 仅执行一次 flush | 硬编码 |
| STO-REG-03 | 文件损坏或 JSON 解析失败时日志 error，以空 Map 启动 | 硬编码 |
| STO-REG-04 | 项目绑定文件 (`data/context.json`) 不存在时自动创建空文件 `{}`，禁止启动报错 | 硬编码 |
| STO-REG-05 | get 不存在的 key 时返回 undefined，禁止抛出异常 | 硬编码 |
| STO-REG-06 | flush 写入时先写 `*.bak` 临时文件再 rename 覆盖原文件；启动时主文件损坏则尝试读取 `*.bak` 恢复 | 硬编码 |
| STO-REG-07 | 轮次计数器为内存级（不持久化），重启后清零；`incrementRound` 返回累加后的计数值 | 硬编码 |

## 4. 功能逻辑（详细步骤 + 生产级伪代码）

### SessionStore 加载与恢复

```python
def load_sessions(file_path):
    # 步骤1: 检查文件是否存在
    if not file_exists(file_path):
        logger.warn("会话映射文件不存在，创建空文件")
        write_file(file_path, "{}")
        return {}

    # 步骤2: 读取文件内容
    raw = read_file(file_path)

    # 步骤3: 尝试 JSON 解析
    try:
        data = json_parse(raw)
        # LANGUAGE_SPECIFIC: TypeScript - 使用 JSON.parse()，catch SyntaxError
    except JSONError:
        logger.error("会话映射文件损坏，尝试读取备份", file_path)
        # 尝试读取 .bak 备份
        if file_exists(file_path + ".bak"):
            try:
                bak_raw = read_file(file_path + ".bak")
                data = json_parse(bak_raw)
                if typeof data == "object":
                    logger.info("已从备份文件恢复会话映射")
                    return data
            except:
                pass
        logger.error("备份文件也损坏或不存在，以空 Map 启动")
        return {}

    # 步骤4: 验证数据结构为对象
    if typeof data != "object":
        logger.error("会话映射格式异常，以空 Map 启动")
        return {}

    return data
```

### SessionStore Set 与防抖写入

```python
def set_session(key, value):
    # 步骤1: 更新内存
    memory_map[key] = value

    # 步骤2: 重置防抖定时器
    if debounce_timer:
        clear_timeout(debounce_timer)

    # 步骤3: 2s 后执行 flush
    debounce_timer = set_timeout(flush, 2000)
    # LANGUAGE_SPECIFIC: TypeScript - 使用 setTimeout

def flush(self):
    # 步骤1: 清除防抖定时器
    debounce_timer = null

    # 步骤2: 序列化内存 Map 为 JSON
    json_str = json_stringify(memory_map)

    # 步骤3: 原子写入 + 备份（rename 顺序不可错）
    tmp_path = file_path + ".tmp"
    bak_path = file_path + ".bak"
    # LANGUAGE_SPECIFIC: TypeScript - 使用 fs.promises.writeFile，utf-8
    write_file(tmp_path, json_str)
    # 正确顺序：先重命名旧文件为 .bak，再重命名 tmp 为当前文件
    # 如果顺序颠倒（tmp→bak 再 tmp→file），第二次 rename 时 tmp 已不存在
    rename_file(file_path, bak_path)   # 现有文件 → .bak（可能失败如果是首次写入，忽略）
    rename_file(tmp_path, file_path)   # tmp → 当前文件（原子覆盖）
    # 进程崩溃时最多丢失内存中最近一次变更，.bak 落后一版
```

### ContextStore 读写

```python
def set_context(session_id, project_id):
    # 步骤1: 更新内存
    memory_map[session_id] = project_id

    # 步骤2: 立即写入（无防抖，因为项目切换频率低）
    json_str = json_stringify(memory_map)
    write_file(context_file_path, json_str)
    # LANGUAGE_SPECIFIC: TypeScript - 使用 fs.promises.writeFile，utf-8

def get_context(session_id):
    # 步骤1: 从内存查询
    project_id = memory_map.get(session_id)
    return project_id  # 不存在返回 undefined

def delete_context(session_id):
    # 步骤1: 从内存删除
    if session_id in memory_map:
        del memory_map[session_id]

    # 步骤2: 立即写入文件
    json_str = json_stringify(memory_map)
    write_file(context_file_path, json_str)

def clear_all_context():
    # 步骤1: 清空内存
    memory_map.clear()

    # 步骤2: 写入空对象
    write_file(context_file_path, "{}")
```

## 9. 内部接口（供其他模块调用）

**接口：`SessionStore.get(key)`**
- **用途**：ai-handler 模块在 `processAIMessage` 中查询已有 sessionId
- **认证**：内部函数调用，无需认证
- **成功响应**：存在返回 `string`，不存在返回 `undefined`

**接口：`SessionStore.set(key, value)`**
- **用途**：opencode 模块创建新 session 后回写映射
- **调用约定**：防抖 2s 后写入文件，不阻塞调用方

**接口：`ContextStore.get(sessionId)`**
- **用途**：index.ts 路由层查询当前会话绑定的项目
- **成功响应**：存在返回 `projectId`，不存在返回 `undefined`

**接口：`ContextStore.set(sessionId, projectId)`**
- **用途**：index.ts 在用户执行「切换项目」指令时写入绑定
- **调用约定**：立即写入，不防抖

**接口：`ContextStore.delete(sessionId)`**
- **用途**：index.ts 在用户执行「重置项目」指令时清除绑定
- **调用约定**：立即写入，不防抖

**接口：`ContextStore.clearAll()`**
- **用途**：测试或全重置场景清空所有项目绑定

**接口：`SessionStore.getRoundCount(key)`**
- **用途**：ai-handler 在 processAIMessage 入口查询当前轮次数，判断是否达到上限
- **返回**：`number`（默认 0）

**接口：`SessionStore.incrementRound(key)`**
- **用途**：ai-handler 每轮消息处理前调用，轮次 +1
- **返回**：累加后的轮次数

**接口：`SessionStore.resetRound(key)`**
- **用途**：ai-handler 在实例回收后调用，将轮次重置为 0
- **调用约定**：与 `sessions.delete(sessionKey)` 配合使用

## 11. 安全要求

- **敏感数据处理**：存储的 sessionId 和 projectId 为内部标识符，不包含用户个人身份信息
- **文件权限**：JSON 文件通过操作系统文件权限保护，建议 600 或 640
- **数据完整性**：写入时异常捕获，不影响内存状态；文件损坏时以空状态启动，不阻塞主进程
