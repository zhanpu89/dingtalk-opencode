## 1. 测试范围

- **被测模块**：消息队列（`src/message-queue.ts`——按 sessionKey 串行化消息处理，Promise 链实现）
- **被测接口**：
  - `enqueue(sessionKey, fn)`
  - `pendingCount()`
- **用例总数**：5 条（单元测试 5）
- **关联业务规则**：BR-QUEUE-001~BR-QUEUE-003
- **关联代码评审报告**：无

## 3. 单元测试用例（UNIT）

### 业务规则覆盖用例

| 用例 ID | 用例标题 | 优先级 | 关联规则 | 前置条件 | 测试步骤 | 预期结果 | 备注 |
|--------|---------|-------|---------|---------|---------|---------|-----|
| TC-QUEUE-UNIT-001 | 依次入队 2 个相同 key——第 2 个在第 1 个完成后执行 | P0 | BR-QUEUE-001 | 空队列 | 1. 创建 MessageQueue 2. enqueue("key1", () => 延迟 10ms 的任务 A) 3. enqueue("key1", () => 任务 B) 4. 等待两个 Promise 完成 | 任务 A 在任务 B 之前执行（串行），任务 B 在任务 A 完成后启动 | |
| TC-QUEUE-UNIT-002 | 第 1 个失败——第 2 个仍执行 | P0 | BR-QUEUE-002 | 空队列 | 1. enqueue("key1", () => 抛出异常的任务 A) 2. enqueue("key1", () => 任务 B) 3. 等待 Promise | 任务 A 抛出异常但不影响链；任务 B 正常执行 | |
| TC-QUEUE-UNIT-003 | 并发入队不同 key——并行执行 | P1 | BR-QUEUE-003 | 空队列 | 1. enqueue("key1", async () => { await delay(50); order.push(1); }) 2. enqueue("key2", async () => { order.push(2); }) 3. 等待 | key2 的任务不必等待 key1 的任务完成（并行），order 中 2 可能在 1 之前 | |
